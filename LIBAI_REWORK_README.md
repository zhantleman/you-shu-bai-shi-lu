# 李白线重构与第四幕小游戏

## ✅ 已完成的功能

### 第一部分：李白线"探索-验证-选择"标准流程

#### 1. 状态与流程定义
- **`libaiState`** 对象包含：
  - `currentAct` = 1 (第一幕)
  - `currentStep` = 'opening' (当前步骤：opening/explore/quiz/choice)
  - `quizScore` = 0 (知识验证答对数)
  - `poetryCount` = 从localStorage读取的诗心值
  - `wineCount` = 从localStorage读取的酒意值
  - `exploredOptions` = 探索选项记录

#### 2. 流程控制
严格按以下顺序推进：
```
opening -> explore -> quiz -> choice -> (进入下一幕)
```

#### 3. 底部按钮与流程绑定

##### a) 初始状态 (opening)
- 页面加载后自动播放李白开场白
- 播放完毕后自动进入探索阶段
- 触发 `enterExploreStep()`

##### b) 探索阶段 (explore)
- **按钮1**："这鲁班机甲如此巨大，师傅可知它有何特别？"
- **按钮2**："这里人山人海，比长安西市还热闹！"
- **按钮3**：隐藏
- 点击任一探索按钮后：
  - 用"累积对话"播放李白反馈
  - 禁用并变灰该按钮
  - 检查两个探索按钮是否都被点击
  - 如都点击，自动进入验证阶段

##### c) 验证阶段 (quiz)
- 播放李白提问："小子，记性如何？方才说的‘鲁班首展’，在何处举办？"
- **按钮1**："在成都IFS七层！"(正确选项)
- **按钮2**："在武侯祠旁边…"(错误选项)
- 点击正确选项：`quizScore += 1`，显示"记性不差！"
- 点击错误选项：显示"哈哈，罚你多看两眼！"
- 答题后自动进入终极选择

##### d) 终极选择阶段 (choice)
- 播放李白提问："那么此情此景，在你心中，是更偏诗篇，还是更近江湖？"
- **按钮1**："此景壮丽，当赋诗一首！"(诗心)
- **按钮2**："走，师傅，咱们寻那酒肉香气去！"(酒意)
- **按钮3**：显示"前往下一站"
- 选择后：
  - 诗心/酒意+1
  - 保存到localStorage
  - 更新页面顶部属性显示

#### 4. 属性显示与存储
- 页面顶部的"诗心"和"酒意"实时更新
- 使用 `localStorage.setItem()` 持久化存储
- 页面加载时从localStorage读取初始值

### 第二部分：第四幕小游戏框架

#### 1. HTML结构
```html
<section id="act4-game" style="display: none;">
  <h2>第四幕：重构机甲诗篇</h2>
  <div id="puzzle-container" class="puzzle-grid">
    <!-- JS动态生成3x3=9个格子 -->
  </div>
  <div>
    <button id="reset-puzzle">重置</button>
    <span id="timer">用时: 0秒</span>
  </div>
</section>
```

#### 2. CSS样式
- `.puzzle-grid`：3x3网格布局
- `.puzzle-tile`：固定宽高、边框、可点击
- 空格子特殊样式
- 计时器显示样式

#### 3. 流程触发
- 点击第三幕"前往下一站"按钮时：
  1. 隐藏剧情容器
  2. 显示第四幕容器
  3. 调用 `initPuzzleGame()`

#### 4. 游戏功能
- **动态生成**：9个格子（1-8编号 + 1个空格）
- **交互逻辑**：点击与空白格相邻的格子可交换位置
- **计时器**：实时显示游戏用时
- **完成判定**：检查格子顺序是否为 [1,2,3,4,5,6,7,8,'']
- **完成奖励**：根据用时给予额外诗心奖励
